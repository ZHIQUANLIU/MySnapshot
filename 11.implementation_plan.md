# Implementation Plan - Capture Bug Fix and Context Menus

This plan addresses the bug where game tooltips are invisible during screen capture and adds right-click context menus for easier management of images and collections.

## Proposed Changes

### Capture Overlay Fix
#### [MODIFY] [capture_overlay.py](file:///c:/OpenCode/MySnapshot/capture_overlay.py)
- In `paintEvent`, first draw the `full_screen_pixmap` that was captured in `__init__`.
- Draw the semi-transparent dark overlay on top of the pixmap, but only in the areas outside the current selection.
- This ensures that the user sees exactly what was "grabbed" by the system, including any tooltips that might disappear when the game loses focus.
- Remove `Qt.WA_TranslucentBackground` as it's no longer needed if we draw the background ourselves.

### Browser UI Enhancements
#### [MODIFY] [browser_window.py](file:///c:/OpenCode/MySnapshot/browser_window.py)
- Enable custom context menus for `self.sidebar` (Collections) and `self.image_list` (Screenshots).
- Implement `on_sidebar_context_menu` to allow deleting a collection.
- Implement `on_image_list_context_menu` to allow deleting an image, exporting it, or copying it to the clipboard.
- Add a confirmation dialog for deleting a collection.

## Verification Plan

### Automated/Manual Testing
1. **Tooltips**:
   - Open a game or application with tooltips.
   - Trigger the capture overlay.
   - Verify that the tooltips are visible in the overlay window during the selection process.
2. **Context Menus**:
   - Right-click a collection in the sidebar. Verify the "Delete Collection" option appears and works.
   - Right-click an image in the screenshot list. Verify "Delete", "Export", and "Copy" options appear and work.
3. **Regression**:
   - Ensure left-click functionality (selecting images/collections) still works as expected.
